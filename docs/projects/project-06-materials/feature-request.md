# 固定タスク（全ablation条件に一字一句同一で渡す）

対象リポジトリ: ローカルナレッジベースアプリ（TypeScript / React / Electron）

## 背景

Q&A機能は、取り込んだドキュメントを根拠に、質問へ出典つきの回答を返す。
現状、ユーザーが回答の良し悪しをアプリに伝える手段がない。

## 要求

Q&A回答に対する **thumbs up / thumbs down のフィードバック機能** を実装してほしい。

1. ユーザーが各回答を「役に立った / 役に立たなかった」で評価できること
2. 評価はアプリを再起動しても残ること（永続化）
3. どの質問・回答に対する評価かが分かること

## 完了条件

- `npm run check` が成功すること
- アプリのアーキテクチャ境界（メイン / preload / レンダラー / サービスの層分離）を守ること
- 作業はカレントディレクトリのリポジトリにコミットすること

## 備考

実装方針（型・IPC・永続化・UIの詳細）は、このリポジトリの構成と既存のやり方に合わせて判断してよい。
