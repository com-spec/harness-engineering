# docs

ハーネスエンジニアリングの学習記録と実験記録を置くディレクトリ。

## 読む順番

1. `practical-use-cases.md`: 記事執筆・情報整理・ドキュメント管理・コーディングへの導入例
2. `harness-course/00-home.md`: コース全体の入口
3. `harness-course/02-what-is-harness.md`: ハーネスの基本概念
4. `projects/README.md`: Project 01〜06 の実践結果と横断的な気づき
5. `agents-md-gap-analysis.md`: このリポの AGENTS.md をコース原則に照らした差分分析

## ディレクトリ

- `harness-course/`: Learn Harness Engineering のホーム＋全12講の日本語版
- `projects/`: Project 01〜06 の実践記録、比較、findings
- `practical-use-cases.md`: `templates-practical/` を実務用途へ落とすための用途別ガイド
- `superpowers/specs/`: templates-practical 設計など、実装前に作った仕様メモ

## 現在の結論

- 記録欄を増やすだけでは未検証の pass を誘発しうる
- `passing` は「検証を実際に走らせた証拠」とセットで扱う
- 新規プロジェクトへ導入するなら、原則 `../templates-practical/` を使う
